# PRD 撰写技能

输出 `docs/prd.html` 单文件 PRD,适用个人项目与小团队工具。**不**用于企业级需求文档。

## 执行步骤

### 1. 确认范围

- 先用 `AskUserQuestion` 问清楚:是个人项目还是要给团队/客户看
- 个人项目走五块骨架;企业 PRD 不在本技能范围
- **严禁**在用户未要求时扩展到非功能需求、数据模型、里程碑——这些归开发文档

### 2. 视觉沿用

- 读取仓库 `design/` 下既有风格指南(如 `stoic.html`)的 CSS 变量、字体、色彩
- PRD 排版用**文档式** h1/h2/h3,**不要**照抄设计稿的 `01 — XXX` section-tag 与卡片堆砌
- 字体色彩沿用,排版自定

### 3. 五块骨架

| 章节 | 内容 |
|------|------|
| 产品定位 | 两句话讲清做什么、与同类的根本分歧 |
| 功能表 | v1 必须/可选清单,表格 + chip 标签 |
| 样机图 | 2×2 HTML 手机壳渲染 4 个核心页 |
| 架构 | 纵向分层 + 底层多源并列 |
| 待决 | 核心未决项,bullet 不带编号,标红 chip |

不写其他章节。

### 4. 左侧大纲

- 240px sticky 侧栏列所有 h2
- IntersectionObserver 监听 `main h2[id]`,最上方可见标题加 active 态
- `html { scroll-behavior: smooth; }` + 每个标题加 `scroll-margin-top: 32px`
- 窄屏(< 900px)折到顶部

### 5. 样机图

- 用 HTML `<div>` 画手机壳,**不**插图片、**不**用 SVG
- `aspect-ratio` 必须先问目标设备:
  - 华为/小米/三星近代旗舰:`9 / 19.5`
  - iPhone 旧款:`9 / 19.5`,刘海屏:`9 / 19.5`
  - 老款 16:9:`9 / 16`
- 内部内容字号缩到 9-11px,沿用主色调

### 6. 凝练原则

- **严禁**口语化和填充词("我们""目前""可能""一些")
- **严禁**数字/中文数字编号承担结构,仅靠标题层级
- **严禁** emoji
- **必须**用 `AskUserQuestion` 收集决策点,不用纯文本开放式提问
- 关键约束/未决项用 chip 或 callout 标识,而非长段散文

### 7. 预览自查

- 写完后 `open docs/prd.html` 触发浏览器预览
- 自查:大纲跳转、滚动联动高亮、手机壳比例、表格对齐
- 用户反馈后**只改不堆**——减内容优先于加内容
