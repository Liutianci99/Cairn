08 — 设计原则

正文固定 640,不响应缩放

视口变宽,只是周围空气更多;阅读体验不变。

Sidebar 是"岛",不是"列"

距正文 ~186px,不参与正文断行决策。

字号层级靠"对比",不靠"倍数"

H1 52 → H2 25 → body 20。落差不均,反而稳。

单位间距 = 32

段间 32 · 章节 ×2 · grid gap 32 — 一个数撑起全篇。

Hero icon 大且圆

192/32 比例,占两列宽。做小会"失重"。

Meta 不带容器

没有边框、没有背景、没有 padding 包裹。

09 — 中文化落地注意点

① H1 字号需要下调

英文 52px / lh 1.2 看起来"大气",中文同尺寸会"压顶"且断行混乱。中文 H1 建议 40–44 / lh 1.3。正文 20px 保持不变。

② 段落间距从 32 → 28

中文行高已经放大到 1.75–1.85,段间再 32 会"散"。建议 p { margin-bottom: 28px }。

③ 正文列宽 640 → 680

英文 640 ≈ 80 chars,中文 640 ≈ 30 字。30 字一行偏窄,容易破"气"。建议中文用 680–720px(约 34–36 字/行)。

④ Sidebar 在窄屏怎么办

"远距 sidebar"在 ≥ 1280px 视口才成立。窄屏建议:

  • 1024–1280px:sidebar 改为 hug,gap 缩到 48–64px
  • ≤ 1024px:meta 折叠到 H1 上方,横向 inline
  • ≤ 720px:meta 全部进入 hero 区,作为标签链

⑤ Hero icon 中文场景偏大

192 在英文里"挑衅且自信",中文阅读者可能觉得"喧宾夺主"。可下调到 144 × 144 · radius 24,保持两列占位比例不变。

"规范不是参考,是承诺。640 不是 max-width,是 640。32 不是大约,是 32。"
